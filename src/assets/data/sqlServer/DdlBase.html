DDL LENGUAJE DE DEFINICIÓN DE DATOS

  1. Sirve para estructurar la tabla, sus definiciones son:
    a. CREATE
    b. DROP
    c. ALTER
  2. Se empieza con las tablas que no tienen relación con otras, es decir aquellas que solo tienen PK luego las que tienen FK.
  3. Si se necesita comentar una línea se utilizan --. Luego se hacen los comentarios
  4. Las FK deben tener el mismo tipo de dato que la PK y nombre de ser posible para evitar conflictos

Crear

  1. CREATE DATABASE BD_FAMILIA
    GO
  2. BD_FAMILIA ES NOMBRE DE LA BASE
  3. no debe empezar con un número o algún carácter especial y tampoco debe contener espacios en blanco.

MODIFICAR

  1. USE MASTER
    GO

    ALTER DATABASE BD_INSTITUTO
    MODIFY NAME=INSTITUTO
    GO
  2. BD_INSTITUTO es el nombre de la base de datos
  3. La modificación se hace en la master, sea el nombre o algún archivo del sistema

ELIMINAR

  1. DROP DATABASE INSTITUTO
  2. INSTITUTO ES NOMBRE DE LA BASE
  3. La eliminación se hace en la master

VER BDs

  1. EXEC SP_HELPDB
  2. Permite ver todas las BD
