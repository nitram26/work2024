FOREIGN KEY

En SQL, la palabra clave CONSTRAINT se usa para definir restricciones en las columnas de una tabla, con el fin de garantizar la integridad de los datos. Una restricción (constraint) puede asegurar que los valores en una columna cumplan ciertas condiciones o relaciones entre tablas.

    CUANDO SE ESTA CREANDO LA TABLA

  1. NOMBRE_TABLA1 es la tabla donde se insertará la FK (COLUMNA) es la columna que contiene la PK que se insertara NOMBRE_TABLA2 es la tabla donde está ubicada la PK

    2. CUANDO YA HA SIDO CREADA LA TABLA

  2. ALTER TABLE NOMBRE_TABLA1
    ADD FOREIGN KEY (COLUMNA) REFERENCES NOMBRE_TABLA2
    GO
    3. AL INICIAR LA TABLA
