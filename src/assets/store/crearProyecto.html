SUBIR UN PROYECTO A GITHUB 

    

      Creado el proyecto se siguen las indicaciones del mismo repositorio de GitHub:

      Paso 1: Crear un repositorio en GitHub
       1. Ve a [github.com](https://github.com/) e inicia sesión.
       2. Haz clic en el botón **New** en la página de inicio o en tu perfil.
       3. Escribe el nombre del repositorio y selecciona si quieres que sea público o privado.
       4. No marques las opciones para agregar archivos, como el README, ya que agregarás tus propios archivos locales.
       5. Haz clic en **Create repository**.
       
     Paso 2: Conectar tu proyecto local al repositorio en GitHub
       1. Abre tu terminal o consola en la carpeta de tu proyecto local.
       2. Inicializa Git en el proyecto (si no lo has hecho antes):
          ```bash
          git init
          ```
       3. Agrega todos los archivos del proyecto al área de preparación:
          ```bash
          git add .
          ```
       4. Haz un primer commit:
          ```bash
          git commit -m "Primer commit"
          ```
       5. Conecta el repositorio local al repositorio de GitHub.
        Copia la URL del repositorio de GitHub (que debería verse como `https://github.com/tu-usuario/tu-repositorio.git`) y luego ejecuta:
          ```bash
          git remote add origin https://github.com/tu-usuario/tu-repositorio.git
          ```
       6. Empuja (sube) tus archivos a GitHub:
          ```bash
          git push 
          o sino
          git push -u origin main
          o también 
          $ git push --set-upstream origin master
          ```
       
       ### Nota
       Si el repositorio en GitHub está configurado para usar `main` como la rama principal, asegúrate de usar `main` en el comando de `push`.
        Si estás usando otra rama (como `master`), ajusta el comando en consecuencia.
       
       ### Paso 3: Verifica en GitHub
       Ve a tu repositorio en GitHub y verifica que los archivos de tu proyecto se hayan subido correctamente.
